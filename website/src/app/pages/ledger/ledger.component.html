What we do NOT collect

  • Workflow definitions, prompts, agent outputs, or any task content
  • User identifiers, tenant IDs, organization names, or API keys
  • Per-workflow events — only a once-daily aggregated rollup

Each deployment that opts in generates a random anonymous installation ID on first boot and reports only counters: workflow runs, tokens invested, and the structural categories of proposals produced. Telemetry is disabled by default and must be explicitly enabled via swarmai.self-improving.telemetry-enabled=true.
